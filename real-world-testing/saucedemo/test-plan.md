# Test Plan – SauceDemo Login Functionality

## 1. Objective
Validate the login functionality of the SauceDemo web application to ensure correct authentication behavior and proper handling of valid and invalid credentials.

## 2. Scope

### In Scope
- Login with valid credentials
- Login with invalid credentials
- Validation of empty fields
- Error message behavior

### Out of Scope
- Backend authentication services
- Security penetration testing

## 3. Test Approach

Testing will include:

- Functional testing
- Negative testing
- Exploratory testing

## 4. Test Environment

Application: SauceDemo  
URL: https://www.saucedemo.com  
Browser: Google Chrome  
OS: Windows 11

## 5. Entry Criteria

- Application must be accessible
- Test accounts must be available

## 6. Exit Criteria

- All test cases executed
- Critical defects documented

## 7. Risks

- Authentication failures
- Session handling issues
- Incorrect error messages
